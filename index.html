Estilização de Titulos com especificidade:

Quando selecionamos no css o h1 mais de uma vez, o ultimo codigo prevalecera, isso chamamos de efeito cascata, porem quando usasmos uma classe com "id" indendente do ultimo codigo digitado, a classe prevalecera, pois tem maior especificidade.

Diferença de especificidade entre CLASSE e ID:

Id:

  • Identificador único
  • Só pode ser usado uma vez por página.
  • Ideal para elementos específicos (ex: um botão único)

Classe

  • Pode ser usada em vários elementos.
  • Serve para aplicar estilos ou comportamentos repetidos.
  • Mais flexível e reutilizável

Qual é mais indicado para usar?

Usar classe no HTML é geralmente melhor porque oferece mais flexibilidade e reutilização no código. Enquanto o id serve para identificar um único elemento, a classe pode ser aplicada a vários, permitindo que estilos e comportamentos sejam compartilhados entre eles. Isso torna o desenvolvimento mais organizado, facilita a manutenção e evita repetições desnecessárias. Em projetos maiores, o uso de classes é essencial para manter o código limpo e eficiente.

Por que é melhor usar classe?

  • Facilita a manutenção do código.
  • Permite aplicar o mesmo estilo a vários elementos.
  • Evita conflitos e limitações que o id pode causar.

Combinadores:

Combinadores são elementos do CSS que definem relações entre os seletores, ou seja, como um elemento está conectado a outro dentro da estrutura do HTML. Eles permitem aplicar estilos com base na posição relativa dos elementos.

Exemplo:

  • Espaço () seleciona descendentes (ex: div p pega todo paragrafo dentro da div )
  • Maior (>) seleciona filhos diretos (ex: ul > li)
  • Mais (+) seleciona o irmão imediato (ex: h1 + p)
  • Tihl (~) seleciona irmãos posteriores (ex: h1 ~ p)

Pseudoclasse:

Uma pseudoclasse é um recurso do CSS usado para aplicar estilos a elementos com base em seu estado ou posição, sem precisar adicionar classes no HTML. Ela permite estilizar elementos quando estão, por exemplo, sendo hovered (passando o mouse), selecionados, ou são o primeiro filho de um pai. Não precisa de uma classe.

Exemplo:

  • p:first-child - seleciona o priemiro filho
  • nth-child(3) aplica o estilo no enésimo filho, nesse exemplo o terceiro.

Box Model/Display

border:

No Box Model do CSS, a border (borda) é a área que envolve o conteúdo e o padding de um elemento. Ela funciona como uma "moldura" ao redor da caixa e pode ter espessura, cor e estilo (como sólida, pontilhada ou tracejada).

  • A border é a borda que envolve o conteúdo e o padding de um elemento no Box Model do CSS.
  • Cria uma moldura visual ao redor do elemento.
  • Espessura: definida em px, em, %, etc.
  • Estilo: sólido (solid), pontilhado (dotted), tracejado (dashed), duplo (double), entre outros.
  • A border aumenta o tamanho total do elemento, exceto quando se usa box-sizing: border-box;

Padding:

  • O padding é o espaçamento interno entre o conteúdo do elemento e sua borda (border) no Box Model.
  • Cria “respiro” dentro do elemento, aumentando a área de fundo (background) sem alterar o conteúdo.
  • Pode ser definido em px, em, rem, %, entre outras unidades.
  • padding-top - espaçamento superior.
  • padding-right - espaçamento à direita.
  • padding-bottom - espaçamento inferior.
  • padding-left - espaçamento à esquerda.

Shorthand

  • padding: 10px; → aplica em todos os lados.
  • padding: 10px 20px; → 10px em cima/baixo e 20px nos lados.
  • padding: 10px 20px 30px 40px; → topo, direita, baixo, esquerda (sentido horário).
