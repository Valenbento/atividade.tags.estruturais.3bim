Estilização de Titulos com especificidade:

Quando selecionamos no css o h1 mais de uma vez, o ultimo codigo prevalecera, isso chamamos de efeito cascata, porem quando usasmos uma classe com "id" indendente do ultimo codigo digitado, a classe prevalecera, pois tem maior especificidade.

Diferença de especificidade entre CLASSE e ID:

Id:

  • Identificador único
  • Só pode ser usado uma vez por página.
  • Ideal para elementos específicos (ex: um botão único)

    • Classe

    • Pode ser usada em vários elementos.
    • Serve para aplicar estilos ou comportamentos repetidos.
    • Mais flexível e reutilizável

    Qual é mais indicado para usar?

    Usar classe no HTML é geralmente melhor porque oferece mais flexibilidade e reutilização no código. Enquanto o id serve para identificar um único elemento, a classe pode ser aplicada a vários, permitindo que estilos e comportamentos sejam compartilhados entre eles. Isso torna o desenvolvimento mais organizado, facilita a manutenção e evita repetições desnecessárias. Em projetos maiores, o uso de classes é essencial para manter o código limpo e eficiente.

    Por que é melhor usar classe?

    • Facilita a manutenção do código.
    • Permite aplicar o mesmo estilo a vários elementos.
    • Evita conflitos e limitações que o id pode causar.
